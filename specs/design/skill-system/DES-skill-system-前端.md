## 页面设计

### 技能管理页

#### 页面结构
- **顶部栏**: 标题"技能管理" + "新建技能"按钮 + 分类筛选下拉框
- **主区域**: 左右分栏布局
  - **左侧（40%）**: 技能列表表格
  - **右侧（60%）**: 技能编辑面板（选中技能时显示）或空状态提示
- **底部**: 技能测试面板（可折叠展开）

#### 交互流程
1. 页面加载 → 调用 `GET /skills` 获取列表
2. 点击列表中某行 → 右侧展示编辑面板
3. 编辑后点击"保存" → 调用 `PUT /skills/{id}`
4. 点击"新建技能" → 右侧切换为创建表单
5. 切换状态开关 → 调用 `PATCH /skills/{id}/toggle`
6. 点击删除 → 确认弹窗 → 调用 `DELETE /skills/{id}`
7. 点击"AI 优化" → 调用 `POST /skills/ai-create` → 结果填充到编辑器
8. 展开测试面板 → 选择 agent + 输入问题 → 调用 `POST /skills/{id}/test` → 展示完整 prompt

#### 组件

- **技能列表**
  - 表格列: 名称、分类（tag 样式）、绑定智能体数量、优先级、状态开关、操作按钮
  - 分类筛选: 下拉框，选项包括"全部"和各分类
  - 内置技能标记: 名称旁显示"内置"badge
  - 选中行高亮

- **编辑面板**
  - 名称输入框（text）
  - 描述输入框（text）
  - Instructions 编辑器（textarea，monospace 字体，约 16 行高度）
  - 分类下拉框
  - 优先级数字输入框
  - 绑定智能体多选（checkbox 列表，从 `GET /agents` 加载）
  - 触发关键词输入（逗号分隔文本）
  - 作用域选择（universal/specific + 数据源多选）
  - "保存"/"取消"按钮
  - "AI 优化"按钮（调用 AI 生成，结果填充到 instructions 编辑器）

- **测试面板**
  - 智能体名称下拉框
  - 数据源名称下拉框（可选）
  - 问题文本框
  - "预览 Prompt"按钮
  - 结果展示区: `<pre>` 标签展示完整系统提示词

#### 接口
- `GET /skills` — 列表加载
- `POST /skills` — 创建
- `PUT /skills/{id}` — 更新
- `DELETE /skills/{id}` — 删除
- `PATCH /skills/{id}/toggle` — 状态切换
- `POST /skills/{id}/test` — 测试预览
- `POST /skills/ai-create` — AI 辅助创建
- `GET /agents` — 加载智能体列表（用于绑定多选）

#### 错误处理
- 创建/更新失败: 弹窗提示错误信息
- 删除内置技能: 按钮禁用，不展示删除选项
- AI 创建失败: 弹窗提示并保持编辑器内容不变
