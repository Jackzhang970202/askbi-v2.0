## 统一聊天页设计

### 页面结构
侧边历史区 → 会话头部 → 消息列表 → 输入区（含加号上下文入口）

### 核心状态

| 状态 | 说明 |
|------|------|
| `currentChatId` | 当前会话 ID |
| `chatContext.type` | `general` / `bi` / `excel` / `team` |
| `chatContext.datasourceName` | 数据源名称 |
| `chatContext.teamId` | 团队 ID |
| `chatContext.teamName` | 团队名称 |
| `selectedSkillIds` | 当前会话使用的技能 |

### 交互
- 点击“新建对话”立即创建普通会话
- 输入区左侧新增“加号”按钮，打开上下文菜单
- 上下文菜单至少包含：选择数据库数据源、选择 Excel 数据源、选择团队、清除当前上下文
- 选择上下文后，在输入区上方或头部展示当前生效标签
- 历史恢复时同步恢复上下文标签与引用对象

### 输入区设计

| 区块 | 内容 |
|------|------|
| 左侧操作区 | 加号菜单、技能选择 |
| 中间输入区 | 文本输入框 |
| 右侧操作区 | 发送按钮 |
| 上方上下文条 | 当前数据源 / 当前团队 / 清除按钮 |

### 历史展示设计

| 会话类型 | 标签 | 标题策略 |
|------|------|----------|
| `general` | 普通对话 | 第一条问题或默认“新对话” |
| `bi` | BI | 数据源名 + 最近问题摘要 |
| `excel` | Excel | Excel 数据源名或文件名 |
| `team` | Team | 团队名 + 最近问题摘要 |

### 路由与恢复
- `#/chat` 进入最近会话或空白普通对话态
- `#/chat/{id}` 打开指定会话，并从后端加载消息与上下文
- 不再要求通过 `activeTab` 控制消息执行类型

### 接口
- `POST /create_chat` 或统一新建会话接口：创建普通会话
- `POST /chat/{id}/context`：挂载或切换上下文
- `POST /chat/{id}/context/clear`：清除上下文
- `POST /ask`：BI
- `POST /excel/ask`：Excel
- `POST /teams/{id}/run`：Team
- `POST /chat/ask` 或等效接口：普通对话

## 团队入口调整

### 规则
- 团队管理页保留，作为配置入口
- 聊天页中的团队不再是“新建页独立模式”，而是当前会话可挂载的一种上下文
- 团队上下文启用后，聊天页头部展示团队名称和清除入口

## 图表分析体验设计

### 目标
将图表结果从“默认图表渲染输出”提升为“聊天中的分析卡片组件”，统一卡片壳层、信息层级、主题色板与配置交互。

### 结果卡片结构

| 区域 | 位置 | 内容 | 约束 |
|------|------|------|------|
| 图表区 | 卡片主体上半部 | 图表画布、必要图例 | 不放长标题和复杂操作 |
| 底部信息条 | 卡片底部 | 图表类型图标、短标题、快捷操作按钮 | 单行、弱操作、可截断 |
| 详情面板 | 展开态或侧边面板 | 表格/图表切换、图表类型、风格、色板、字段映射、显示选项 | 表单分组展示 |

### 视觉层级

| 元素 | 视觉权重 | 设计规则 |
|------|----------|----------|
| 主数据图形 | 高 | 线宽 2px、数据点小尺寸、颜色来自主题色板 |
| 卡片选中态 | 高 | 使用单一 ring 高亮，不叠加多重阴影或描边 |
| 标题信息 | 中 | 卡片态单行短标题，详情态可编辑完整标题 |
| 图例 | 中低 | 顶部居中或靠上布局，小圆点样式，避免默认方块图例 |
| 坐标轴/网格线 | 低 | 小字号、低对比、细线、减少存在感 |
| 操作按钮 | 低 | ghost / icon-only / hover 强化 |

### 主题色策略

| 项目 | 规则 |
|------|------|
| 色板来源 | 使用固定预设主题色组 |
| 默认主题 | 首选低冲突、适合业务分析的冷色或中性主题 |
| 序列配色 | 同图表内保持饱和度与明度一致 |
| 禁止项 | 不允许按数据序列随机取色 |

### 图表卡片样式约束

| 项目 | 规则 |
|------|------|
| 外层容器 | `rounded-xl` 级圆角，带边框与轻阴影 |
| 内边距 | 图表区与信息条分别维护，不共用混乱 padding |
| 标题位置 | 标题默认放在底部信息条，不压入绘图区 |
| 图表高度 | 卡片态固定中等高度，详情态允许更高画布 |
| 选中反馈 | 通过外层 ring 或边框强化当前卡片 |

### 详情面板结构

| 分组 | 字段 |
|------|------|
| 基础 | 标题、图表类型 |
| 风格 | 圆润/锐利、渐变开关、主题色 |
| 字段映射 | 横轴、纵轴、颜色、堆叠方式 |
| 显示选项 | 图例、数据标签、坐标轴、网格线 |
| 视图切换 | 表格 / 图表 |

### 交互规则

1. 图表卡片默认展示精简态，只暴露查看数据与调整配置两个快捷操作。
2. 图表详情中支持表格/图表切换，便于验证数据与图形是否一致。
3. 图表类型切换时只更新图表配置，不改变消息原始数据。
4. 主题色和视觉风格切换应立即反馈到当前图表预览。
5. 长标题在卡片态必须截断，完整标题在详情面板中编辑。
6. 同一消息下多个图表时，当前选中卡片需要有明确主强调态。

## 设计约束

1. 最小改动优先，尽量复用现有聊天页和 Sidebar 布局
2. 发送路由必须依赖会话上下文，不得继续依赖 `activeTab` 作为主判断
3. Team、BI、Excel 均保留原有执行能力，只调整入口与状态模型
4. UI 必须允许用户在同一会话内完成普通对话 → 绑定上下文 → 切换上下文 → 清除上下文
5. 图表体验改造优先解决卡片层级、色板统一、标题与操作分层，不追求一次性引入复杂编辑器能力
