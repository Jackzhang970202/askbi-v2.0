## REQ-skill-system-技能管理

**版本**: v1.0.0 | **状态**: 未开始 | **优先级**: P0

### 需求描述
系统提供可配置的技能（Skill）管理功能，技能本质上是一组提示词指令（instructions），可以在运行时动态注入到智能体的系统提示中，增强或约束智能体的行为。管理员可以创建、编辑、删除、启用/禁用技能，并为技能配置分类、优先级、绑定智能体和触发关键词。

### 前置条件
- 用户已登录且具有 admin 或 manager 角色
- 数据库已创建 `askbi_skills` 表
- 系统启动时已种子内置技能

### 输入
- 技能名称、描述、instructions（Markdown 文本）
- 分类（general/sql/report/chart/analysis/excel）
- 绑定智能体列表（JSON 数组）
- 触发关键词列表（JSON 数组）
- 优先级（0-100）
- 作用域（universal/specific + 数据源列表）

### 输出
- 技能列表（支持分类筛选）
- 技能详情
- 操作结果（创建/更新/删除/切换状态）

### 处理规则
1. 创建技能时名称不可重复
2. 内置技能（`is_builtin=TRUE`）不可删除，但可编辑 instructions 和切换启用状态
3. 技能按优先级降序排列注入到智能体 prompt
4. 技能作用域为 `specific` 时仅对指定数据源生效
5. 技能绑定特定智能体时仅注入到该智能体，未绑定则注入到所有智能体

### 验收标准
- [ ] 可创建新技能并保存到数据库
- [ ] 可编辑技能的 instructions 内容
- [ ] 可删除非内置技能
- [ ] 内置技能不可删除但可禁用
- [ ] 技能列表支持分类筛选
- [ ] 技能启用/禁用状态正确持久化

## REQ-skill-system-技能注入

**版本**: v1.0.0 | **状态**: 未开始 | **优先级**: P0

### 需求描述
当工作流执行 LLM 调用时，系统自动根据当前智能体名称和数据源，加载匹配的活跃技能，将技能的 instructions 拼接到系统提示词中，实现动态 prompt 增强。

### 前置条件
- 存在已启用的技能
- 工作流正常执行中

### 输入
- 智能体名称（如 `bi_sql_agent`）
- 数据源名称（可选）

### 输出
- 增强后的系统提示词（base_instructions + skill instructions）

### 处理规则
1. 加载所有 `is_active=TRUE` 的技能
2. 按 binding_agents 过滤：空数组匹配所有智能体，非空则检查当前智能体是否在列表中
3. 按 scope_type 过滤：`universal` 匹配所有数据源，`specific` 检查当前数据源
4. 按 priority 降序排列
5. 拼接格式：`{base_instructions}\n\n## 附加规则\n{skill1_instructions}\n\n{skill2_instructions}`
6. 技能查询结果缓存 60 秒（TTL），避免每次 LLM 调用都查询数据库

### 验收标准
- [ ] 活跃技能在 LLM 调用时被正确注入到系统提示词
- [ ] 技能绑定过滤逻辑正确
- [ ] 技能作用域过滤逻辑正确
- [ ] 无活跃技能时使用原始系统提示词不变
- [ ] 技能缓存机制生效（60s TTL）

## REQ-skill-system-AI辅助创建

**版本**: v1.0.0 | **状态**: 未开始 | **优先级**: P1

### 需求描述
用户可通过描述技能的目标功能，调用 LLM 自动生成技能的 instructions 内容，降低编写高质量提示词的门槛。

### 前置条件
- 用户已登录且具有 admin 或 manager 角色

### 输入
- 技能功能描述（自然语言）
- 技能分类

### 输出
- LLM 生成的 instructions 文本

### 处理规则
1. 将用户描述和分类作为上下文发送给 LLM
2. LLM 输出结构化的 Markdown 格式 instructions
3. 返回给前端供用户预览和编辑

### 验收标准
- [ ] AI 辅助生成的 instructions 内容合理且结构化
- [ ] 生成结果可在编辑器中二次编辑

## REQ-skill-system-技能测试

**版本**: v1.0.0 | **状态**: 未开始 | **优先级**: P1

### 需求描述
用户可在技能管理界面预览某个技能注入到指定智能体后的完整系统提示词，验证注入效果。

### 前置条件
- 技能已创建

### 输入
- 技能 ID
- 智能体名称
- 数据源名称（可选）
- 测试问题（可选）

### 输出
- 完整的系统提示词文本
- 注入的技能 instructions 文本

### 处理规则
1. 模拟技能加载流程，返回拼接后的完整 prompt
2. 不实际执行 LLM 调用

### 验收标准
- [ ] 测试面板正确展示注入后的完整 prompt
- [ ] 技能 instructions 在 prompt 中正确拼接
