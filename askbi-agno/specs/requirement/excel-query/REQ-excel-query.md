## REQ-excel-query-文件上传

**版本**: v1.0.0 | **状态**: 已完成 | **优先级**: P0

### 需求描述
用户上传一个或多个 Excel 文件，系统保存原始文件到会话目录，执行预处理拆表，并记录拆分结果元数据。

### 前置条件
- 用户已登录

### 输入
- chat_id（可选，为空时自动生成）
- 一个或多个 Excel 文件
- sub_name_rows（可选）
- table_header_rows（可选）

### 输出
- chat_id
- 文件元数据（文件名、保存路径列表）

### 处理规则
1. 未提供 chat_id 时自动生成（`excel_` 前缀）
2. 原始文件保存到 `runtime/excel_chats/{chat_id}/`
3. 按配置执行预处理拆表
4. 创建/更新会话，`datasource_name=__excel__`
5. 保存通用元数据与文件配置
6. 推送上传事件到进度服务
7. 文件预览同时支持原始文件与处理后文件

### 验收标准
- [ ] 文件保存到正确的会话目录
- [ ] 文件名使用唯一标识避免冲突
- [ ] chat_id 自动生成格式正确
- [ ] 元数据正确保存到数据库
- [ ] 多文件上传正确处理

## REQ-excel-query-自然语言分析

**版本**: v1.1.0 | **状态**: 未开始 | **优先级**: P0

### 需求描述
用户通过自然语言提问，系统生成 Python/pandas 代码分析上传的 Excel 文件，并按统一回答结构返回问数结果，默认包含问题结果/结论、回答依据/佐证、数据图表；当用户勾选“分析解读”时，额外输出业务视角的分析解读。

### 前置条件
- 用户已上传 Excel 文件到会话
- 文件存在于会话目录中

### 输入
- chat_id
- 自然语言问题
- 是否输出分析解读（可选开关，默认关闭）

### 输出
- `summary`: 主回答正文，按统一结构组织
- `code`: 生成的 Python/pandas 代码
- `result`: 代码执行结果
- `chart`: ECharts option 或 `{"chart_needed": false, "reason": "..."}`
- `trace`: 过程元数据、执行信息、报告与图表数据

### 输出结构要求
1. **问题结果、结论**
   - 必须先直接回答用户问题
   - 对统计、排名、波动、异常、最值等问法需明确给出结论
2. **回答依据、佐证**
   - 必须包含支撑结论的明细数据、聚合结果或详细数据表
   - 可补充必要计算口径或推理说明
3. **数据图表**
   - 系统需基于结果数据自主选择合适图表
   - 若用户指定图表形式，优先满足指定形式
   - 不适合做图时允许返回不出图原因
4. **分析解读（可选）**
   - 仅在用户勾选时输出
   - 内容为业务视角分析，不重复结论与依据
5. **建议模块**
   - 可不单独成段
   - 如需输出，可融合进分析解读

### 处理规则
1. 从会话目录获取 Excel 文件路径列表
2. 加载文件元数据（文件名、sheet、列名、样例数据）
3. 构建 prompt，要求 LLM 生成 pandas 分析代码
4. 调用 LLM 生成 Python 代码并清理 markdown 包裹
5. 在受限环境中执行代码（提供 `pd/json/os/FILE_LIST/FILE_METADATA`）
6. 序列化执行结果，处理 NaN、DataFrame 等对象
7. 调用 LLM 生成主回答 `summary`，并按“结果/结论 → 依据/佐证 → 图表说明 → 可选分析解读”的顺序组织
8. 调用 LLM 生成 `chart`
9. 保存会话消息与请求记录，历史消息中的 `structuredData` 需保留 `summary/code/result/chart/trace`

### 验收标准
- [ ] 生成的代码能正确读取会话内文件
- [ ] 代码执行结果正确
- [ ] 执行失败时返回错误信息
- [ ] `summary` 首段直接回答问题结果或结论
- [ ] `summary` 包含可支撑结论的详细数据/佐证部分
- [ ] 有合适数据时生成图表，无合适数据时返回 `chart_needed: false`
- [ ] 未勾选分析解读时不输出该部分
- [ ] 勾选分析解读时输出业务视角解读
- [ ] 消息与记录正确保存

## REQ-excel-query-会话管理

**版本**: v1.1.0 | **状态**: 已完成 | **优先级**: P0

### 需求描述
支持 Excel 问数会话的列表查看、消息查看、文件数据获取与删除。

### 前置条件
- 用户已登录

### 输入
- 列表: 无
- 消息: chat_id
- 文件数据: chat_id
- 删除: chat_id

### 输出
- 列表: Excel 会话列表（仅 `excel_` 开头会话）
- 消息: 消息列表（含结构化数据）
- 文件数据: 文件预览（文件名、sheet、列、前 50 行数据）
- 删除: 成功/失败

### 处理规则
1. 列表过滤仅 Excel 类型会话（chat_id 以 `excel_` 开头）
2. 历史消息中的结构化数据字段需与提问接口返回保持一致
3. 文件数据获取优先从会话目录读取，回退到元数据中的 `excel_paths`
4. 区分原始文件与处理后文件（`processed__` 前缀）
5. Excel 文件按 sheet 分别展示数据，CSV 文件直接读取
6. 删除时清理数据库记录、进度缓存与会话目录

### 验收标准
- [ ] 列表仅返回当前用户的 Excel 会话
- [ ] 文件数据正确展示（列名、前 50 行）
- [ ] Excel 多 sheet 正确分离展示
- [ ] 删除会话后文件与数据已清理

## REQ-excel-query-从数据源初始化

**版本**: v1.0.0 | **状态**: 未开始 | **优先级**: P1

### 需求描述
从已有的 Excel 数据源复制文件到新的问数会话，快速启动分析。

### 前置条件
- 存在有效的 Excel 类型数据源

### 输入
- chat_id（可选）
- datasource_name

### 输出
- chat_id
- 文件元数据

### 处理规则
1. 验证数据源存在且为 Excel 类型
2. 从数据源配置获取 `file_dir` 和 `files`
3. 复制文件到新的会话目录
4. 同时复制 `processed__` 版本（如果存在）
5. 创建会话并保存元数据

### 验收标准
- [ ] 从数据源正确复制文件到新会话
- [ ] 会话创建成功
- [ ] 元数据正确保存

## REQ-excel-query-进度推送

**版本**: v1.0.0 | **状态**: 未开始 | **优先级**: P1

### 需求描述
Excel 问数任务执行过程中，前端可实时获取结构化事件进度。

### 前置条件
- 用户已发起 ask 请求

### 输入
- chat_id

### 输出
- 事件列表（时间、事件类型、消息）

### 处理规则
1. Workflow 通过 `progress_callback` 推送事件
2. 事件类型至少包括：`started`、`metadata_loaded`、`code_started`、`code_round`、`report_completed`、`chart_completed`、`completed`
3. 每个事件附带时间戳与消息内容
4. 前端通过 `/excel/progress?chatid=` 获取事件列表
5. 流式展示文本以 `structuredData.summary` 为准

### 验收标准
- [ ] 事件按执行顺序返回
- [ ] 每个事件包含时间、类型、消息
- [ ] 完成事件标记任务结束

## REQ-excel-query-文件下载与保存

**版本**: v1.0.0 | **状态**: 进行中 | **优先级**: P2

### 需求描述
支持 Excel 问数会话中文件的下载与修改保存（当前为占位实现）。

### 前置条件
- 会话已存在

### 输入
- 下载: chatid, filename（可选）, is_modified（可选）
- 保存修改: payload（文件数据）
- 保存原始: payload（文件数据）

### 输出
- 下载: `{status, chatid, filename, is_modified}`
- 保存: `{status, payload}`

### 处理规则
1. `/excel/download_file` 返回下载信息（当前为占位响应）
2. `/excel/save_modified_file` 接收修改后的文件数据（当前为占位响应）
3. `/excel/save_original_file` 接收原始文件数据（当前为占位响应）
4. 三个接口当前均返回简单成功响应，实际文件操作待后续实现

### 验收标准
- [ ] 下载接口返回正确响应格式
- [ ] 保存接口接收数据并返回成功
- [ ] 后续实现完整文件读写逻辑
