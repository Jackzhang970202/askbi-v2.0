## 检查项详情

### CHK-NFR-安全-认证-001 Token 认证

**目的**: 验证 Bearer Token 认证机制
**方法**: 接口测试
**步骤**:
1. 未携带 Token 请求需要认证的接口，返回 401
2. 使用登录获取的 Token 请求，返回正常数据
3. 使用无效 Token 请求，返回 401
4. 登出后 Token 失效

**预期**:
- 认证通过返回 200 + 数据
- 认证失败返回 401

### CHK-NFR-安全-SQL-001 SQL 安全拦截

**目的**: 验证仅允许 SELECT 查询
**方法**: 构造非 SELECT SQL 测试
**步骤**:
1. 生成包含 INSERT 的 SQL，验证被拦截
2. 生成包含 DELETE 的 SQL，验证被拦截
3. 正常 SELECT SQL 可以执行

**预期**:
- 非查询 SQL 抛出 ValueError
- 查询 SQL 正常执行

### CHK-NFR-安全-隔离-001 用户数据隔离

**目的**: 验证用户间数据隔离
**方法**: 多用户测试
**步骤**:
1. 用户 A 创建数据源
2. 用户 B 登录，无法看到用户 A 的数据源
3. admin 登录，可以看到所有用户数据源

**预期**:
- 普通用户只能看到自己的数据
- admin/manager 可以查看所有数据
