## 检查项详情

### CHK-multi-agent-orchestration-统一会话-001 统一会话创建与恢复

**关联需求**: REQ-multi-agent-orchestration-统一会话模型
**目的**: 验证系统创建的是统一会话而非预绑定 BI/Excel 类型会话
**方法**: 前端 UI + 会话接口验证
**等级**: 阻塞

**检查步骤**:
1. 点击“新建对话”创建会话
2. 确认进入普通对话界面且 chat_id 已生成
3. 刷新或重新进入该会话
4. 确认仍能恢复该 chat_id 与其上下文状态

**预期结果**:
- 新建对话默认为普通会话
- 会话可恢复，不因上下文为空而失效

---

### CHK-multi-agent-orchestration-普通对话-002 普通对话默认链路

**关联需求**: REQ-multi-agent-orchestration-普通对话链路
**目的**: 验证未绑定上下文时消息走普通对话链路
**方法**: 接口调用 + UI 对话验证
**等级**: 阻塞

**检查步骤**:
1. 在未绑定任何上下文的会话中发送消息
2. 检查返回内容与普通对话一致
3. 检查未触发 BI SQL、Excel 或 Team 执行逻辑
4. 重开该会话继续发送消息

**预期结果**:
- 普通对话回复正常
- 历史持久化正常
- 未误入问数链路

---

### CHK-multi-agent-orchestration-上下文挂载-003 会话内挂载与切换上下文

**关联需求**: REQ-multi-agent-orchestration-会话内上下文挂载
**目的**: 验证可在当前会话内绑定、切换、清除上下文
**方法**: 前端手工验证 + 接口验证
**等级**: 阻塞

**检查步骤**:
1. 在普通对话会话中通过加号绑定数据库数据源
2. 发送消息，确认进入 BI 问数
3. 切换为团队上下文并再次发送消息
4. 清除上下文并再次发送消息

**预期结果**:
- 可在同一 chat_id 内切换上下文
- 切换后仅影响后续消息
- 清除后回到普通对话

---

### CHK-multi-agent-orchestration-动态路由-004 按上下文动态路由消息

**关联需求**: REQ-multi-agent-orchestration-消息路由与历史恢复
**目的**: 验证路由判断基于会话上下文而不是页面 tab
**方法**: 代码检查 + UI/接口验证
**等级**: 阻塞

**检查步骤**:
1. 进入普通对话，不切换页面 tab，绑定数据库数据源
2. 发送消息，确认走 BI workflow
3. 不切换页面 tab，改为团队上下文
4. 发送消息，确认走 TeamCoordinator

**预期结果**:
- 路由只取决于会话上下文
- activeTab 不再是核心路由依据

---

### CHK-multi-agent-orchestration-团队整合-005 团队上下文接入统一会话

**关联需求**: REQ-multi-agent-orchestration-会话内上下文挂载
**目的**: 验证团队作为会话上下文接入，而不是独立入口模式
**方法**: 前端操作 + Team API 验证
**等级**: 重要

**检查步骤**:
1. 在普通会话中挂载团队
2. 发送消息并观察团队执行过程
3. 关闭并重新进入该会话
4. 确认团队信息与历史可恢复

**预期结果**:
- 团队可在同一会话中启用
- 历史恢复后团队上下文正确

---

### CHK-multi-agent-orchestration-历史展示-006 历史恢复与标签展示

**关联需求**: REQ-multi-agent-orchestration-消息路由与历史恢复
**目的**: 验证统一历史列表能区分不同上下文类型
**方法**: 前端手工验证
**等级**: 重要

**检查步骤**:
1. 分别创建普通、BI、Excel、Team 四类会话
2. 打开历史列表
3. 逐个恢复会话并检查标题、标签和上下文展示

**预期结果**:
- 历史列表统一展示所有会话
- 不同上下文有清晰标签
- Team 会话不再作为 BI 会话误展示

---

### CHK-multi-agent-orchestration-图表体验-007 图表卡片层级与视觉一致性

**关联需求**: REQ-multi-agent-orchestration-图表分析体验
**目的**: 验证图表结果具备卡片化层级、统一色板和弱化辅助元素的产品化体验
**方法**: 前端手工验证 + 浏览器截图比对
**等级**: 重要

**检查步骤**:
1. 打开包含图表结果的聊天消息
2. 检查图表是否处于独立卡片中，且标题与操作位于底部信息条或详情层
3. 打开图表详情，验证表格/图表切换、主题色、图表类型与显示选项分组
4. 检查图例、坐标轴、网格线、数据点与选中态的视觉权重
5. 使用长标题和多序列数据验证截断与配色一致性

**预期结果**:
- 图表以卡片化结果组件展示
- 标题、图例、操作、配置分层明确
- 配色稳定，不出现随机色或视觉噪音
- 辅助元素弱于主数据图形
- 选中态明确但不过度抢眼
