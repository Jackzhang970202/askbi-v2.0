## 检查项详情

### CHK-auth-登录-001 登录功能

**关联需求**: REQ-auth-登录登出
**目的**: 验证登录功能
**方法**: API 测试
**等级**: 阻塞

**检查步骤**:
1. 正确用户名密码登录，验证返回 token 与用户信息
2. 错误用户名密码，验证返回 401
3. 使用返回的 token 请求需要认证的接口，验证通过
4. 验证 token 在 TOKEN_CACHE 中存在

**预期结果**:
- 登录成功返回 token
- 登录失败返回 401
- Token 可用于认证

### CHK-auth-权限-004 RBAC 权限控制

**关联需求**: REQ-auth-RBAC权限
**目的**: 验证角色权限控制
**方法**: 多角色测试
**等级**: 阻塞

**检查步骤**:
1. admin 调用用户管理接口，验证成功
2. user 调用用户管理接口，验证返回 403
3. admin 查看所有用户数据，验证通过
4. user 查看其他用户数据，验证被过滤

**预期结果**:
- admin 拥有全部权限
- user 仅可查看自己的数据

### CHK-auth-默认用户-005 默认 admin 账户

**关联需求**: REQ-auth-登录登出
**目的**: 验证默认 admin 账户自动创建
**方法**: 启动测试
**等级**: 阻塞

**检查步骤**:
1. 新数据库启动服务
2. 使用 admin/admin123 登录
3. 验证登录成功

**预期结果**:
- 默认 admin 账户存在
- 密码为 admin123

### CHK-auth-前端状态-007 前端认证状态管理

**关联需求**: REQ-auth-登录登出
**目的**: 验证前端 token 管理与请求拦截
**方法**: 端到端测试
**等级**: 阻塞

**检查步骤**:
1. 登录成功，验证 token 存储
2. 请求 API，验证自动附加 Authorization header
3. 登出，验证 token 清除
4. 请求 API，验证 401 自动跳转登录

**预期结果**:
- Token 正确存储与清除
- 请求自动携带 token
- 未登录自动跳转
