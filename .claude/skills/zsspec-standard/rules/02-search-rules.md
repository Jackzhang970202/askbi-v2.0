# 检索最小化规则

**版本**: v0.1

---

## 核心原则

禁止在未收敛目标文件范围前执行大范围搜索。

## 逐级获取上下文

必须按以下顺序获取上下文：

| 顺序 | 步骤 | 说明 |
|------|------|------|
| 1 | 用户明确文件 | 先读用户明确提到的文件、目录或已知入口文件 |
| 2 | 单个子目录 | 再读单个明确子目录 |
| 3 | 逐级扩大 | 仅当以上信息不足时，才允许逐级扩大范围 |
| 4 | 说明理由 | 每次扩大范围前，必须能说明上一轮为什么不够 |

## 优先入口文件

| 类型 | 文件 |
|------|------|
| 前端项目 | `package.json` |
| Java项目 | `pom.xml` |
| 主入口 | `main.*`、`App.*` |
| 配置文件 | `application*.yml` |
| 已有文档 | `specs/` 目录下文档 |
| Brain结论 | 当前模块已有的 brain 结论 |
| 模板文件 | `.claude/skills/zsspec-standard/` |

## 检索自检清单

执行搜索前必须检查：

- [ ] 是否存在更精确的文件或目录？
- [ ] 是否可能扫入大体量依赖/产物目录？
- [ ] 当前目标是否真的需要目录遍历？
- [ ] 是否已按"文件 → 子目录 → 模块目录"逐级尝试？
- [ ] 这次搜索是否只是为了确认某目录或文件不存在？（如是，默认不得搜索）

若任一问题答案不明确，不得执行大范围搜索。

## 禁止的高风险搜索

| 禁止模式 | 说明 |
|----------|------|
| `**/*` | 递归所有文件 |
| `**/src/**/*` | 递归 src 目录 |
| 根目录扫描 | 以项目根目录为起点的大范围扫描 |
| 未限定搜索 | 无边界、可能命中依赖目录的全文搜索 |

## 高风险目录

搜索时必须排除：

- `node_modules/`
- `target/`
- `dist/`
- `build/`
- `.git/`
- 缓存目录
