# 数据模型规范

**版本**: v0.1

---

## 命名规范

### 表命名

| 类型 | 规则 | 示例 |
|------|------|------|
| 系统表 | sys_{表名} | sys_user |
| 业务表 | biz_{模块}_{表名} | biz_order_info |

### 字段命名

| 类型 | 规则 | 示例 |
|------|------|------|
| 主键 | id | id |
| 外键 | {实体}_id | user_id |
| 状态 | {实体}_status | order_status |
| 布尔 | is_{描述} | is_deleted |

### 索引命名

| 类型 | 规则 | 示例 |
|------|------|------|
| 唯一 | uk_{字段} | uk_user_name |
| 普通 | idx_{字段} | idx_create_time |

## 通用字段

| 字段 | 类型 | 可空 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | BIGINT | 否 | 自增 | 主键 |
| create_time | DATETIME | 否 | NOW | 创建时间 |
| update_time | DATETIME | 否 | NOW | 更新时间 |
| deleted | TINYINT | 否 | 0 | 逻辑删除 |

## 字段类型

| 类型 | 使用场景 |
|------|----------|
| BIGINT | 主键、外键 |
| INT | 计数、状态码 |
| VARCHAR(n) | 字符串 |
| TEXT | 长文本 |
| DATETIME | 日期时间 |
| DECIMAL | 金额 |

## 索引原则

- 每个表必须有主键
- 业务唯一字段建唯一索引
- 根据查询条件创建索引
- 单表索引不超过5个


## SQL文件命名

| 类型 | 规则 | 示例 |
|------|------|------|
| 数据库迁移 | {描述}_YYYYMMDDHHmmss | create_user_table_20260416143000 |

**原则**: 代码目录中生成的 SQL 文件必须以时间戳结尾，格式为 `YYYYMMDDHHmmss`（年月日时分秒），确保文件命名唯一且可追溯执行顺序。

## SQL 同步原则

- 项目已有后端 SQL 目录 → 同步生成到该目录；无 → 生成到项目根目录 `sql/` 文件夹，命名：`{表名}_{时间戳}.sql`（如 `sys_user_20260512.sql`）
- SQL 文件须与模块设计文档中的数据表设计保持一致，设计字段变更时同步更新对应 SQL
- 变更 SQL 时若不确定更新方式，需让用户确认：全量同步或增量更新
