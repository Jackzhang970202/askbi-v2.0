# AskBI 升级开发说明

## 1. 升级目标

本次开发的核心目标是把 AskBI 从“多入口能力集合”升级为“统一会话驱动的分析平台”。

| 升级点 | 开发目标 |
|------|------|
| 会话统一 | 所有能力共享同一会话模型 |
| 路由统一 | 由会话上下文决定执行链路 |
| 能力统一 | Skill、智能体、记忆模块接入统一能力体系 |
| 结果统一 | 文本、过程、表格、图表形成结构化交付 |
| 记忆升级 | 引入 Mem0 支撑用户画像与会话摘要记忆 |

---

## 2. 主要开发内容

| 模块 | 开发内容 |
|------|------|
| 会话模型 | 增加普通对话、BI、Excel 与 Skill 增强态支持；支持上下文挂载、切换、清除、恢复 |
| 聊天交互 | 新建对话默认进入普通对话；输入区增加上下文挂载入口；历史记录统一展示与恢复 |
| 执行链路 | 普通对话接入默认链路；BI、Excel 根据当前上下文动态路由；Skill 作为增强层附着在主链路上 |
| 能力管理 | Skill 管理、智能体管理、记忆配置进入统一产品能力体系 |
| 记忆模块 | 引入 Mem0，沉淀用户画像与会话摘要记忆，支持读取、写入与控制 |
| 图表体验 | 图表结果升级为卡片化分析组件，支持表图切换、主题色、风格、字段映射与显示项配置 |

---

## 3. 开发顺序建议

| 阶段 | 开发重点 | 目标 |
|------|------|------|
| 第一阶段 | 会话模型与接口 | 先打通统一会话和上下文读写 |
| 第二阶段 | 前端聊天页与历史恢复 | 让新建、切换、恢复形成完整闭环 |
| 第三阶段 | 普通对话、BI、Excel 与 Skill 增强路由 | 让所有消息都按上下文正确执行 |
| 第四阶段 | Skill、智能体与 Mem0 记忆整合 | 完成能力配置、记忆沉淀与挂载体系 |
| 第五阶段 | 图表卡片与分析体验升级 | 提升分析结果展示与汇报效果 |

---

## 4. 验收重点

| 验收项 | 标准 |
|------|------|
| 会话一致性 | 同一会话内可挂载、切换、清除不同能力，且历史不丢失 |
| 历史恢复 | 点击历史后可正确恢复消息、上下文、输入区和执行语义 |
| Skill 生效 | Skill 选择后可对当前主能力产生增强作用，并可恢复 |
| 记忆有效性 | Mem0 中的用户画像与会话摘要可被正确写入、读取与控制 |
| 执行正确性 | 普通对话、BI、Excel 都能按当前上下文正确执行 |
| 图表体验 | 图表卡片完整、配置有效、主题统一、支持表图双视图 |
| 产品完整性 | 用户能在一个聊天入口内完成从提问、挂载能力、读取记忆、查看结果到调整图表的完整闭环 |
